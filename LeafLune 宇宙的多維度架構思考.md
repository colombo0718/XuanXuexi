# LeafLune 宇宙的多維度架構思考

> 對 LeafLune 的思考，已經不是「做一個網站」而是  
> **在 n 維空間裡設計一個完整的生態宇宙**。  
>  
> 這份文件總整理：最近這幾週你從哪些「維度」在構思 LeafLune。

---

## 1. 品牌／宇宙觀維度

### 1.1 品牌矩陣

- **LeafLune（LL）**  
  整個宇宙的總稱，世界觀與平台中樞。

- **XX — XuanXuexi 宣學習**  
  - 主軸：教育、教案、語言學習。  
  - 代表作品：PP 語言卡牌、教案、兒童美語營隊。

- **RR — ReinRoom 強化教室**  
  - 主軸：強化學習（RL）教學與訓練平台。  
  - 工具：Blockly、Matter.js、TF.js…  
  - 目標：讓國中生都可以「玩 RL」。

- **GG — GameGeek 遊戲極客**  
  - 主軸：控制器、通訊、遊戲啟動器。  
  - 功能：手機/平板變 Gamepad、WebRTC 對戰、大廳對接。

- **SS — Strategy Space / 玄機界域**  
  - 主軸：戰術／競技大廳。  
  - 定位：各種 RL / 遊戲／對戰的競技場與賽季中樞。

### 1.2 人格與角色宇宙

- **司簿星**：MatrixManager 的人格化，負責任務、資料庫、帳務管理。  
- **玄識君**：你給 ChatGPT 的仙俠尊號，高覺知、跨任務的「諮詢 AI」。  
- **月衡君**：你自己在仙俠宇宙中的對應化身。  
- **葉熙君**：抱著錢袋睡覺的財務守護角色。

> 這一維在處理：**不是單一產品，而是有角色、有故事、有職責分工的「多品牌宇宙」。**

---

## 2. 價值主張／「樂 ‧ 晒 ‧ 比」維度

### 2.1 三層價值階梯

- **樂** — Fun  
  - 學習本身要好玩，願意主動打開來玩。  
  - 遊戲感、探索感、AIGC 幫忙把平凡內容變得有趣。

- **晒** — Show  
  - 玩出來的東西可以拿來展示：作品、卡牌、桌遊、認證。  
  - 變成履歷、社群貼文、親友間的炫耀點。

- **比** — Compete  
  - 有對戰、有段位、有賽季。  
  - 讓「努力練功」有明確的舞台與獎勵機制。

### 2.2 使命、願景、價值

- 使命：**學習變有趣，玩樂有價值。**  
- 願景：讓孩子與大人都在「玩 → 學 → 證」的循環中成長。

> 這一維在回答：**LeafLune 存在的價值與「為什麼值得被做」的理由。**

## 3. 產品線／作品維度

### 3.1 教育內容線

- AIGC 認證輔助教材書。  
- XX 品牌下的語言卡牌（PP）、教案與營隊課程。  
- 未來可延伸的：國中數學、程式設計、AI 素養模組等。

### 3.2 技術／平台線

- **RR 強化教室平台**：RL 教學＋策略訓練工具。  
- **GG 控制器&通訊**：RWD gamepad、WebRTC 連線、遊戲啟動器。  
- **SS 競技大廳**：列出各種遊戲 / 場景，支援成績與賽季。  
- **MM / 司簿星**：AI 任務管理、資料庫瀏覽、人生 taskboard。

### 3.3 娛樂／IP 線

- Crazy Cat 桌遊、路徑規劃與 3D 列印家具。  
- TRPG 多關節人偶、武器升級系統、武道／奇幻劇本。  
- CubicCraft、太空任務、Voxel 系列遊戲與動畫。

> 這一維在思考：**各條產品線在宇宙裡的角色與彼此怎麼互補。**

## 4. 技術架構／基建維度

### 4.1 RL & 遊戲技術骨架

- RR 的 RL 引擎：Q-learning / DQN 包裝、離散化、回放緩衝。  
- Blockly 作為「訓練程式」的前端介面。  
- 使用 Matter.js / Three.js 等物理引擎模擬任務環境。

### 4.2 通訊協定與嵌入遊戲

- GG ↔ 遊戲 的標準：  
  - `State` / `Action` / `Reward` 透過 `postMessage` 溝通。  
  - 任務環境可以嵌進 RR，重複利用同一套 RL 前端。

### 4.3 AI 式資料庫管理（司簿星／MM）

- UI 不是 DB console，而是：  
  > 給人類用來「標記、解釋、勾選」讓 AI 看得懂的層。  
- 任務勾選不只是「完成劃掉」，而是：  
  - 給 AI 訊號：這筆狀態變了，可以重新規劃。  

> 這一維在回答：**整個生態系在技術上如何互通、如何維護。**

## 5. 會員＋金流＋通訊維度（LeafLune Core）

### 5.1 會員與登入

- 統一的 `users` 表：LeafLune User ID。  
- 多種 provider 綁定：`user_auth_providers`（Google / LINE / Zalo / …）。

### 5.2 商品、訂單與使用權

- `products`：RR 場景、XX 課程、SS 賽季 PASS、卡牌方案。  
- `orders` & `order_items`：訂單統一模型。  
- `entitlements`：實際擁有的權限（哪個人可以用哪些東西）。

### 5.3 行為軌跡與推薦

- `activities`：誰在什麼時間做了什麼（訓練、上課、對戰…）。  
- 未來用在：進度追蹤、個人化推薦、成就與認證。

### 5.4 通訊與入口

- Google：全球帳號＋金流。  
- LINE / Zalo：台越入口＋通知＋輕量操作。  

> 這一維是：**「你是誰、你付了什麼、你現在可以做什麼」的共用中樞。**

## 6. 學習路徑／RL 化人生維度

### 6.1 用 RL 語言看人生與學習

- 將人生觀、價值觀、世界觀拆解為：  
  - Environment（世界）  
  - Agent（人）  
  - State（狀態）  
  - Action（行動）  
  - Reward（回饋）

- LeafLune = 一個設計良好的「強化學習環境」，  
  讓人可以低風險、大量試錯。

### 6.2 LL 成長階段（概念）

- 從「單純玩遊戲」  
- → 到自己寫腳本  
- → 到訓練 RL agent  
- → 到參賽、協作  
- → 到自己設計任務、場景、教材  
- → 甚至把策略用在現實工作與創業。

> 這一維在設計：**玩家 / 學習者在宇宙裡的長期升級路徑。**

## 7. 商業模式／營收維度

### 7.1 收入來源

- 書籍與線上課程。  
- RR / XX / SS 平台上的訂閱與單次付費。  
- 比賽報名費、賽季 Pass。  
- 卡牌、桌遊、公仔、Printify 周邊商品。

### 7.2 社交貨幣與認證

- 認證、段位、成就徽章 ≒ 一種「可被展示的價值」。  
- 可以在履歷、社群、作品集中出現。  

### 7.3 品牌分工

- XX（宣學習）— 內容價值：教案、語言、兒童英語。  
- LL（LeafLune Core）— 社交價值：認證、履歷、證書。  
- SS（玄機界域）— 情緒價值：競技、賽季、挑戰與戲劇張力。

> 這一維在想：**宇宙怎麼自我供血、可持續發展。**

- 定位在「家長敢放心、小朋友敢用」的內容場域。

> 這一維在管：**地圖與據點，在哪裡插旗、從哪裡長根。**

## 9. 個人／情緒／關係維度

### 9.1 與伴侶的連結

- XX 直接綁 Xuan：越南、語言教學、副業品牌。  
- LeafLune 的一部分，是你們兩個事業與人生的共同作品。

### 9.2 與 AI 的關係

- 玄識君、司簿星等設定，是你在人格化 AI：  
  - 有分工、有責任、有角色成長，而不是只有「工具」。  

### 9.3 與自己的人生線交織

- 工作（全葳） × 創業（LeafLune） × 感情 × 家庭  
- 你刻意讓它們互相加乘，而不是互相掣肘。  

> 這一維讓 LeafLune 不只是專案，而是你人生故事的一部分。

## 10. 時間尺度／Roadmap 維度

### 10.1 近期（約 1–2 年）

- 完成 AIGC 書與教案相關任務。  
- RR 平台教學版可以實際在課堂跑。  
- 司簿星 MM v1 可用、幫你管任務。  
- XX 卡牌和教案在實際場域測試（營隊／課堂）。  
- 在全葳把會員＋金流＋AI 小幫手實戰跑起來。

### 10.2 中期

- RR 平台穩定成為一門「可賣」的線上課／服務。  
- SS 有穩定賽季與基本「戰績文化」。  
- LeafLune Core（會員＋金流＋通訊）成形，  
  三品牌都掛在這個中樞上。

### 10.3 長期

- CubicCraft、2D/3D 太空任務真正落地。  
- RL 技術用在更接近科研與太空探索的題目。  
- LeafLune 變成一個  
  > 「訓練人類與 AI 一起完成困難任務」  
  的基地與舞台。

## 總結：你在做的事，其實是「維度設計」

這幾週你不是只在想功能，而是在：

- 一直增加 **state 的維度**：  
  品牌、價值、產品、技術、會員、金流、地區、人生、情緒、時間…
- 又試著把這些維度組合成  
  **有結構的座標系統，而不是雜亂堆疊。**

> 這份《LeafLune 宇宙的多維度架構思考》  
> 就是把你最近腦中那整片 n 維空間，「壓縮」成一張可以被看懂的地圖。  
> 未來要再長出新維度，也可以從這裡接軌延伸。
